Controller Component

Home | Model | View | Controller

Definition

The Controller receives user requests, communicates with the Model, and sends the results to the View.

Responsibilities

  • Receive user input
  • Validate requests
  • Communicate with the Model
  • Select the appropriate View
  • Return the response to the user

Workflow

  1. User clicks a button.
  2. Controller receives the request.
  3. Controller asks the Model for data.
  4. Model processes the data.
  5. Controller sends the data to the View.
  6. View displays the result to the user.
